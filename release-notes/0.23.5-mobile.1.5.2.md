**0.23.5** is a small follow-up to 0.23.4. Mock-injection coexistence with real provider data — both Mac and the iOS companion (1.5.2 Build 108) needed paired fixes.

iOS pairs with **1.5.2 Build 108** on TestFlight. Update both for the fix to take full effect.

### What's fixed

- **iOS — real accountless providers no longer hidden by mock injection.** When mock injection was on, real provider entries with no email (Claude / Ollama / Copilot) were getting filtered out as "pre-OAuth orphans" because the mock sibling had a synthetic email. Both filter rules now compute thresholds from real entries only; mocks bypass orphan filtering. Build 94's original orphan cleanup for real-vs-real cases is fully preserved.

- **Mac — stranded mock records auto-clean across process restarts.** `lastPushedRecordNames` was in-memory only, so mock CKRecords pushed by a previous Mac process became permanent ghosts on iOS after toggling mocks off + restarting Mac. Mac now reconciles with CloudKit at startup so the next push cycle deletes any stranded records. Generalises beyond mocks — also handles non-mock orphans (e.g. provider disabled before restart).

- **iOS — Raw Sync Data shows 30-day cost + account email per row.** Quick visual diagnostic for multi-device sync issues without drilling into detail.

### Notes

- Mac and iOS are paired — please update both.
- iOS 1.5.2 Build 108 is the same code as Build 107 + updated in-app release notes copy. If you already updated to 107, the data fixes are already live; Build 108 is purely a release-notes refresh.

下面是 0.23.4 (Mobile 1.3.1) 的完整 release notes，作为本次累积变更的上下文。

---

**Hotfix** on top of 0.23. Fixes a Codex JSONL parser bug that silently misattributed most Codex token usage to `gpt-5` regardless of which model was actually running — Cost-tab Daily Spend numbers were undercounted as a result. Every existing user is automatically corrected on first launch; no manual action required.

Mobile companion stays at **1.3.1**. iOS pairs with **1.5.0 Build 100** on TestFlight.

### What's fixed

- **Codex token attribution.** The Codex JSONL scanner had a 32 KB single-line size cap that pre-dated Codex CLI 0.125's larger `turn_context` events (which now bundle the project's `AGENTS.md` / `user_instructions` / `developer_instructions`, ~38–41 KB per line). Every `turn_context` was silently dropped → the parser never saw the active model name → all subsequent token counts fell through to a `gpt-5` default. Cap raised to match the Claude / Pi scanners' 256 KB; the cache parser version bumped in lockstep so every existing on-disk cache file is invalidated on next launch and re-scanned with correct attribution. Cost numbers immediately reflect the real model mix (gpt-5.5 / gpt-5.4 etc. instead of being mostly buried under gpt-5).

- **Cache fingerprint now covers pricing values, not just model names.** A same-name reprice (existing model gets a new rate) used to leave stale baked-in costs in the per-session cache because the fingerprint only hashed the model-name list. Fingerprint now embeds the full price tuple per model — input / output / cache-read / cache-creation / threshold and above-threshold rates — so any pricing edit invalidates every user's cache automatically on the next launch.

- **iOS account merging across Mac versions for non-ASCII emails.** The iOS app's fallback synthesis for legacy account-email identifiers used a different normalization than what the Mac side has been writing since 0.23 (NFC + percent-encoding + length cap). For accounts whose email contained non-ASCII characters (`café@example.com`), the two paths produced different bytes and the same logical account split into two cards on iOS when one Mac was on 0.23+ and another was on an older release. Both sides now share a single normalization module backed by a paired contract test. Ships on iOS **Build 100**.

### Notes

- **CFBundleVersion** = `58.4.1.3.1` (was `58.1.3.1` for 0.23). Sparkle on 0.23 prompts the 0.23.4 update on next check-for-updates.
- iOS 1.3.1 / 1.5.0 (Build 96 / 98) users see the corrected Cost numbers automatically once Mac finishes the post-upgrade rescan and pushes to CloudKit. **iOS 1.5.0 Build 100** ships the matching identifier-normalization fix.
- First launch on 0.23.4 runs a fresh full scan (10–60 s depending on JSONL volume) before the new numbers settle in. After that, refresh is instant as usual.

**0.23.4 是叠在 0.23 之上的 hotfix**。修复一个 Codex JSONL 解析器 bug —— 大部分 Codex token 用量被错误归到 `gpt-5`，不管用户实际跑的是什么模型 —— 直接导致 Cost tab 的 Daily Spend 数字偏低。**所有现有用户**首次启动 0.23.4 后会自动重算修正，无需手动操作。

Mobile companion 仍为 **1.3.1**。iOS 同步 **1.5.0 Build 100** 上 TestFlight。

### 修复内容

- **Codex token 归属**。Codex JSONL 解析器有一个 32 KB 单行长度上限，这个值是早于 Codex CLI 0.125 的产物 —— CLI 0.125 把项目的 `AGENTS.md` / `user_instructions` / `developer_instructions` 都打包进 `turn_context` 事件，单行 ~38–41 KB。每条 `turn_context` 都被静默丢弃 → 解析器永远拿不到真实使用的 model 名 → 后续所有 token 计数落到 `gpt-5` 兜底。这版把上限提到 256 KB（跟 Claude / Pi 解析器一致）；缓存解析器版本同步 bump → 所有用户磁盘上现有的 cost cache 文件都会在下次启动时失效、用修复后的解析器全量重扫。Cost 数字立即按真实模型组合显示（gpt-5.5 / gpt-5.4 等正常出现，不再被 gpt-5 掩盖）。

- **缓存指纹现在覆盖价格，不只是 model 名字**。之前的指纹只 hash 模型名清单 —— 同名 reprice（同一模型新单价）下 per-session cache 里 baked 的成本数字会 stale 残留。修复后指纹包含每个模型完整价格元组（input / output / cache-read / cache-creation / threshold 与 above-threshold 各档），未来任何定价改动都会让所有用户 cache 在下次启动自动失效重扫。

- **iOS 跨 Mac 版本账号合并 —— 非 ASCII 邮箱**。iOS 端对老快照合成 legacy account-email 标识符时用的归一化方式跟 Mac 端从 0.23 起写入的（NFC + percent-encode + 长度 cap）不一致。邮箱里有非 ASCII 字符时（`café@example.com`），两边产出不同字节，同一账号会被切成两张卡 —— 一台 Mac 跑 0.23+、另一台跑老版本时发生。两边现在共用同一个归一化模块，配 paired 契约测试 pin 死字节相等。**iOS Build 100 ship**。
